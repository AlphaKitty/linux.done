❓ 常见问题

我为什么需要这个插件？

LINUX DO 每天新帖很多，真正跟你有关系的大概就那么几个——比如有人提到你关注的工具、分享了你要的资源、或者聊到了你会的话题。

这个插件就是你的"私人雷达"：你设好关键词规则后，它 7×24 帮你盯着论坛。匹配到帖子立刻浏览器弹通知，你不用一遍遍手动刷新，也不会错过想参与的话题。

为什么我明明设了规则，有些帖子却没通知？

可能有以下几种情况，可以逐一排查：

① 你已经看过这个帖子了。插件会自动跳过你已经点开过的帖子，避免重复打扰。

② 规则只选了类别，没选标签或关键词。仅按类别匹配范围太宽，插件要求至少选一个标签或一个关键词才能生效。

③ 同一规则在冷却期内。同一条规则触发后，默认 10 分钟内不会再次对同一类帖子发通知，防止刷屏。

④ 帖子发布时间太久了。如果设置了保留策略，超过设定时长的旧帖会被自动过滤。

⑤ 规则被禁用了。检查规则卡片左侧的开关是否为开启状态。

我需要一直开着 LINUX DO 页面吗？

打开 LINUX DO 页面时：Content Script 会注入页面，直接监听论坛自带的 WebSocket 实时推送。新帖子一到就匹配你的规则，几乎秒级通知。此模式下后台轮询完全跳过，不产生额外请求。

关闭所有 LINUX DO 页面时：Content Script 不存在了，后台 Service Worker 接管——每 10 分钟轮询一次最新帖子列表，匹配到规则就发通知。你已经看过的帖子会被自动跳过，不会重复打扰。

这个插件会对 LINUX DO 网站和我的设备造成很大负担吗？

不会。插件的设计目标就是"几乎零负担"：

对 LINUX DO 网站而言——开着页面时，数据来自页面自身的 WebSocket 推送，不产生额外 API 请求；后台轮询 10 分钟才发一次请求，且走缓存 + 延迟策略，绝不会对服务器造成压力。

对你的设备而言——Service Worker 只在轮询瞬间唤醒（几秒即休眠），Content Script 只在你打开论坛时才注入，匹配逻辑是简单的字符串比较，CPU 和内存占用可以忽略不计。
